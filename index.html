Project 1: 3.1.3 Combo Menu Customization Program

Course: Computer Science Essentials | Date: Fall 2025

Project Objective

The objective of this project was to design and implement an interactive, automated terminal application that handles customer orders for a sandwich shop. The program calculates complex conditional pricing, handles user custom requests (like drink and fry size upgrades), validates input strings, and applies dynamic discount logic for combo meals.

My Responsibilities

  • Programmed structured conditional statements (if/elif/else structures) to cross-reference menu strings with float values.
  • Implemented global tracking variables to maintain state data across custom function structures like cost math algorithms.
  • Developed validation control loops (while True) ensuring user interaction strings are filtered using string mutation routines like .lower() and .strip().
  • Coded dynamic system modifiers to evaluate if multi-tier combos were present, modifying base order floats dynamically to subtract package discounts.

Skills Exemplified & What I Learned

  • Data Type Manipulation: Gained experience explicitly converting variables using float(), int(), and handling raw numerical strings gracefully via .isdigit() checks.
  • Scope & Architecture: Learned how to leverage functions, loops, and global/local scope rules to manage multi-tiered user configurations cleanly.
  • Defensive Programming: Mastered defensive programming concepts by building fall-backs that intercept invalid text inputs and safely prompt the user again without breaking the console runtime loop.

Project 2: 3.1.5 Rock Paper Scissors Algorithmic Challenge

Course: Computer Science Essentials | Date: Fall 2025

Project Objective

The goal of this project was to design a predictive automation bot for a competitive Rock, Paper, Scissors game challenge. By analyzing historical matchup patterns and past round outcomes, the bot dynamically shifts through three distinct multi-tiered defensive counter-strategies to actively disrupt predictable or repetitive opponent behaviors.

My Responsibilities

  • Programmed complex pattern tracking using Python list slice indexing ([-1] and [-2]) to extract real-time context from structural match arrays.
  • Architected recursive conditional boolean checks (lost_last_round and lost_two_rounds_ago) to map multi-turn history trends.
  • Designed a predictive state-machine with three fallback tiers: an anti-spam routine to defeat hardcoded opponent behaviors, a mirroring strategy for round losses, and an optimal outcome matrix for round victories.
  • Implemented structural boundary checks ensuring the system executes gracefully across early-stage rounds where historical tracking variables are null.

Skills Exemplified & What I Learned

  • Algorithmic Thinking: Mastered the translation of basic game logic into dynamic heuristic decision trees that adapt based on external data trends.
  • Array Data Indexing: Developed a functional understanding of working with history strings and lists, accessing past states safely without creating index-out-of-range runtime errors.
  • State Machine Management: Learned how prioritizing rule sequences can change system priorities dynamically based on changing operational conditions.
